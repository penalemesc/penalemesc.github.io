Carl-o-Gotchi with a battery!

Project Status:
Hardware: Completed
Software: Completed *
While the basic software is done (as it is the one given in the tutorial that is linked further below), I have plans to keep updating it in the future, potentially to make something akin to a Digi-vice where the creature inside evolves after a certain set of objectives are completed, however that remains to be seen

Now you may be wondering, what is a Carl-o-Gotchi and why am I adding a battery to it? Well a Carl-o-Gotchi is a project created by Aaron Heidgerken-Greene, Instrument Project Manager and Technical Director of the [Carleton College] Class of '69 Makerspace. It is meant as an instructional project for the Makerspace that teaches students how to solder and to be potentially used as a platform for game development

In regards to the second question of why I added a battery to it, that would be because the current project (as specified in the tutorial for creating it, which can be found here), the model presented does not have battery capabilities because the board that connects everything together is not fully populated with the required circuitry.


Thus having built the original model over the summer, this term I decided I wanted to finally give it a battery, in this article you will find both a full tutorial on how to make it and add the battery and an exploration on how the experience on how I added the battery and all the modifications that I had to do in order for it to work, along with plenty prototypes! The one that should be auto playing at the moment next to this is an extra that I built to gift to someone who said that they wanted the dog to be named Buddy and that is why it does not have the normal "Carl-o-Gotchi" text on boot.

    Instructions for the basic board!

  1. Put the buttons in the spots laid out for them in the custom PCB and solder them from the back
  2. From the side where the clicking part of the button are, put the resistance (on what would be the space for it on the opposite side of where the marking for it is), making it so that the legs go to the same side as the legs of the buttons, twist them so that they are easier to solder. Solder them and cut off the excess with the cutter, making it as close to the solder joint as possible, as the beeper is going to go on top of it
  3. Put the beeper in the space laid out for it, solder it from the opposite side and if there is noticeable excess cut it off

  1. If you haven't, open the protective film of the ESP32S2 and grab 2 of the rows of pins and put them in the custom PCB in the outermost rows for both the bottom side and top side (another explanation: there are 4 rows for pins, put them in the ones that are the furthest from each other), making sure that the black separator that they have is in the same side of the board as where we put the solder connections for the buttons and that the longest side of the pins go towards where the clicking part of the buttons are and solder them from this last side (so the one with the clickers)
  2. With a pair of cutters remove the excess pin that is left after soldering, as close to the solder point as possible as this way we get a better look on the bottom side and so that the screen can be properly placed on the board, which is coming next
  3. (If the screen does not come with its pins soldered to it already then) Solder the short pins to the screen, so the ones that are short from the black divider, making sure they are parallel to the screen or as much as they can as this might affect the fit of the whole thing (this is why its good that a lot of the time these screens come with the pins soldered to them)
Side view of assembled components
Assembled components from the top right

  1. Put the screen on the custom PCB in the row of pins that are directly on top of the buttons, with the screen being on the same side as the buttons, so the long pins go into the board so that the pins are in the same area as the pins for the buttons and solder
  2. With the cutter, cut off the excess from putting on the screen, again as small as possible since the ESP32S2 will go on top of it
  3. Put the ESP32S2 on top of the pins that we had laid out towards the start, making sure that we have the custom PCB placed on a flat surface with the screen down on the surface, with this we place the ESP32S2 with the USB C connector on the left side of the custom PCB (following the lines that are laid out in the board essentially)
  4. Solder the board to the pins

With this we are done with the hardware side of the basic board and we can move on to the software and you may be wondering, shouldn't we be doing the battery? And the answer is NO! The reason for that is that we want to do software first as that way we can make sure that we have not burnt any connections when we were soldering and the only way to do this is through uploading the software! It is important that you do not connect the board to anything yet, for now leave it as is until it is mentioned that you can/should connect it to the computer

    Instructions for software!

  1. Download the code from the Github as a zip file and unzip it in a folder of your choosing. Say you chose to unzip inside of your documents folder, create a folder called Tamogotchi_Basic and move all the unzipped files inside, this is where our project will be hosted since that is the name of the folder that the project was created on and is needed to run the project from Arduino
  2. If you have not yet, install the Arduino IDE, just install everything that it chooses to install unless you know otherwise
  3. With the Arduino IDE open you want to go into the libraries section where we will download and install 2 Libraries
    • ArduinoJson library by Benoit Blanchon
    • TFT_eSPI library by Bodmer
  4. Go to the documents folder of your computer, go into libraries, go into TFT_eSPI, rename the "User_Setup.h" file to "User_Setup_Default.h" and from the Github project copy the file that is named "User_Setup.h" onto that folder

ArduinoIDE is recommended for ease of upload, but other environments work too!
Software working after assembly
  1. With Arduino IDE open, connect the USB Cable to your computer not to the board, press the On button on the board, you are going to want to keep holding this button as you plug in the cable to the board at least until you have verified the next step, the reason for that is that this action puts the board into download mode, which allows us to upload programs onto it!
  2. Once it is connected make sure that Arduino IDE has detected it (if you are on a Mac you might be asked if you want to connect to that, click yes), on the top left of the Arduino IDE you will see a dark drop down menu, it should have text that says some variation of ESP32S2 or Lolin S2 mini, etc, there are plenty options, but essentially you want the name of a board there and it should be some variation of the one we are using
  3. Click the upload button (the arrow that is to the left of where we checked for the board)
    4. Note: Once it finishes uploading an error might appear if it is "Upload error: Failed uploading: uploading error: exit status 1" that is fine and normal, you should not stress about it


We are now technically done with software, if you want to grab the MAC address for the board (for WI-FI purposes for example in the case of Carleton, where you need it for network registration) then you can go onto the Arduino terminal and scroll all the way up and MAC should be listed there.
However if you want to make sure (and you should make sure), click the RST button that is on the opposite side of the board from the On button, it says RST below it. If everything went on correctly then the screen will start displaying the background of the project along with the name that we chose for our Carl-o-Gotchi (essentially what is shown when I turn it on towards the end of the video at the start of this page), it should take a couple of seconds, if it takes a couple of minutes this is likely that the WI-FI is not properly set up as there is code that needs it so I would try to set it up again and download the code onto the board again.

    Instructions for adding a Battery

    As a note: henceforth whenever the ESP32S2 is mentioned unless specified, it will be referring to all the electronics, this is done for simplicity since the ESP32S2 is the main brains of the device and all of it is connected on one single body
  1. If the battery has a special connector at the tip of the cables, remove it, the cables should have the electrically conducting part of the wire at the tip
  2. Look at the charging board and familiarise yourself with where the B+ and B- connections are, you can make a mark on the side of the board to signal one (I recommend marking for the B+ since that is the first one we are going to solder)
  3. Turn the board around so that the charging port is now on top of the table, it is important that the placement of the battery and the board is the same as the one on the picture that is on this section, but essentially the charging board should be close to you on the table or soldering station and the battery should be on the table or a surface, with the cables being on top of the board without soldering them yet
  4. Solder the red cable that comes out of the battery to the B+ connector on the board
  5. Solder the black cable that comes out of the battery to the B- connector on the board
Battery circuitry
ESP32S2 reference from official page
  1. With the board now done with we can set up the slider switch so that we can turn the Carl-o-Gotchi off and on, as a placeholder put the ESP32S2 on the case and the slider switch on the side of the ESP32S2, don't solder anything, measure the distance from one of the outer pins of the switch (this one will indicate On! For your design) to the VBUS connector in the ESP32S2, it is labelled under the board but since it is already soldered, there is a screenshot next to this text (which was taken from the official documentation found here), measure the distance and cut the cable afterwards
  2. With the slider switch being kept where it is, as a placeholder place the charging board on top of the ESP32S2, with the wire connectors of both touching (again this is a placeholder), we will measure 2 distances:
    • The distance from the OUT- connector on the charging board (you can realise its it because it has the black cable of the battery right next to it) to the ground connector on the ESP32S2 that we soldered a pin to before
    • The distance from the OUT+ connector on the charging board (you can realise its it because it has the black cable of the battery right next to it) to the middle pin in the slider switch
  1. Prepare the red cables for soldering by cutting it to length based on the distances we just measured, with the wire strippers, peel the tips of the wire, and with the soldering iron, solder a bit of tin onto these tips so that none of the individual wires are separated
  2. Put some electrical tape on the ESP32S2 where the slider switch is going to go
  3. With the red cable that is meant to go to the middle pin on the slider (shorter one) from OUT+, solder it to the charging board (on OUT+) and then to the middle pin
  4. With the red cable that is meant to go to the side pin on the slider switch, solder it to the VBUS pin on the ESP32S2 and then to the side pin we chose. With a small piece of electrical tape, cover the connection on the ESP32S2 with it so that we have an easier time soldering the next step and for future proofing
  5. With the black cable that is meant to go from the OUT- connector on the charging board to the ground pin on the ESP32S2, solder it to the connector on the changing board and then to the ground pin on the ESP32S2
  6. The charging board is now 'floating' on top of the ESP32S2 and the battery should be on top of the screen, with the hot glue gun, put a little bit on top of the USBC connector on the ESP32S2 and place the charging board USBC port on top, making sure that no hot glue goes into the inside of either port so that we can connect both
The full circuitry of the prototype done from a back bottom view

IMPORTANT NOTE: NEVER CONNECT THE ESP32S2 TO A COMPUTER (in case you want to change the software) WHILE THE SLIDER SWITCH IS ON THE ON POSITION, THIS IS LIKELY TO CAUSE A BURNT ESP32S2 WHICH WE DO NOT WANT AND ALSO IF YOU WANT TO CHARGE IT ONLY CONNECT TO THE CONNECTOR OF THE CHARGING PORT WHICH WHEN ON THE CASE IS THE TOP ONE AS THE ESP32S2 CONNECTOR WILL NOT CHARGE IT, IT WILL ONLY GIVE IT POWER

    Instructions for the case!

  1. With the front part of the case, put it face down on a table so that we can put the buttons of the case inside
  2. Put the buttons in the case, so that when you turn the case over and look at the front the letters in the buttons go A, B, C; lift the case a little so that you can actually fit the buttons into their slots, make sure they are all level and in their holes
  3. Place the electronics inside, making sure that the case buttons did not move
  4. With the back plate of the case, bring it down near the rest of the things and put the battery in between the little bridge that is floating and the back side and bring the case down until it all lines up with the front plate and the electronics fit
  5. Screw the back plate to the front plate

Congrats!

You have just finished your very own "Carl-o-Gotchi" you are now invited to modify the software as much as your heart desires, make some new games! Change the sprites in some way! Add functionality to the beeper (because that has not been coded yet)!
Now you might be thinking, Conrado, I read this whole tutorial and the only mention of a prototype was a single image before a warning, how did you prototype? What did you prototype?
Well the answers to these questions are simple, I had to first prototype a way to get the new electronics connected, to do this I asked Aaron about how to do it, he pointed me to a student worker at the makerspace who had experience with these technologies and this student told me a similar version of what is written here, though to to corroborate I also did a google search where after exploring multiple articles where people were wondering the same question, I arrived at the same conclusion that the student had given me so it became a case of figuring out how to place the new electronics on the case.
It is the case that the original design for the case is not mine, it was created by Aaron and I really liked it because he designed it with a battery in mind even if it would then not be added until me, however what happened with this case is that due to the added height brought on by the charging board, the electronics would not fit inside of it, both because there was no hole for the slider switch but also because the electronics were much to tall.
Thus what I did was go through a couple of iterations of backer case, where I would slowly modify it based on how I had set up the electronics inside. I must also admit that even though the idea might not have gotten across before, it is the case that I had to reorganise the insides a couple of times, for example, originally the PCB of the charging board was on top of the USB connector of the ESP32S2 rather than how it is now where the ports of both are almost touching. So both parts were sort of built at the same time, as I moved the electronics the case was modified a bit and as I figured out the case the electronics were moved, it essentially built on itself.

Interactive 3D Model of the case

I think the most important part of this project it is that it was fun to build and prototype! I think that while there were things that potentially were frustrating, such as the couple of hours I spent racking my brain around why my board would not turn on anymore or even connect to my laptop when it was working perfectly fine the day before, when in reality what had happened is that I had burnt the ESP32S2 by connecting it to my computer at the same time as the battery was on. Which is why now there is a big note about it!
But even that which was a bit frustrating was fun in the end because it led me to experiment on different possibilities and do a lot of research on why things might stop working so yeah I would rate this project a 10/10! Hopefully in the future I can come back to it and create new sprites and other fun additions to it!
