Carl-o-Gotchi with a battery!

Project Status:
Hardware: Completed
Software: Completed *
While the basic software is done (as it is the one given in the turorial that is linked further below), I have plans to keep updating it in the future, potentially to make something akin to a digi-vice where the creature inside evolves after a certain set of objectives are completed, however that remains to be seen

Now you may be wondering, what is a Carl-o-Gotchi and why am I adding a battery to it? Well a Carl-o-Gotchi is a project created by Aaron Heidgerken-Greene, Instrument Project Manager and Technical Director of the [Carleton College] Class of '69 Makerspace. It is meant as an instructional project for the Makerspace that teaches students how to solder and to be potentially used as a platform for game development

In regards to the second question of why I added a battery to it, that would be because the current project (as specified in the tutorial for creating it, which can be found here), the model presented does not have battery capabilites because the board that connects everything together is not fully populated with the required circuitry.


Thus having built the original model over the summer, this term I decided I wanted to finally give it a battery, in this article you will find both a full tutorial on how to make it and add the battery and an exploration on how the experience on how I added the battery and all the modifications that I had to do in order for it to work, along with plenty prototypes! The one that should be autoplaying at the moment next to this is an extra that I built to gift to someone who said that they wanted the dog to be named Buddy and that is why it does not have the normal "Carl-o-Gotchi" text on boot.

    Tutorial
    Materials needed
  • USB C cable
  • A pair of cutters
  • Soldering Iron
  • Tin lead solder wire
  • 1 Beeper
  • 1 150Ω resistor
  • Thin electrical wire, 2 colours recommended, can be any, though black and red are encouraged
  • Battery controller
  • Lipo battery
  • 1 ESP32S2 Mini controller
  • 1 Custom PCB
  • 1 240x240 TFT screen
  • 3D printed case
  • 3 tact switches
  • 1 Small slider switch
  • 2 screws
    Software based materials
  • Arduino IDE
  • Carl-o-Gotchi basic program
    Instructions for the basic board!
  1. Put the buttons in the spots layed out for them in the custom PCB and solder them from the back
  2. From the side where the clicking part of the button are, put the resistance (on what would be the space for it on the oposite side of where the marking for it is), making it so that the legs go to the same side as the legs of the buttons, twist them so that they are easier to solder. Solder them and cut off the excess with the cutter, making it as close to the solder joint as possible, as the beeper is going to go on top of it
  3. Put the beeper in the space layed out for it, solder it from the opposite side and if there is noticeable excess cut it off

  1. If you haven't, open the protective film of the ESP32S2 and grab 2 of the rows of pins and put them in the custom PCB in the outermost rows for both the bottom side and top side (another explanation: there are 4 rows for pins, put them in the ones that are the furthes from eachother), making sure that the black separator that they have is in the same side of the board as where we put the solder connections for the buttons and that the longest side of the pins go towards where the clicking part of the buttons are and solder them from this last side (so the one with the clickers)
  2. With a pair of cutters remove the excess pin that is left after soldering, as close to the solder point as possible as this way we get a better look on the bottom side and so that the screen can be properly placed on the board, which is comming next
  3. (If the screen does not come with its pins soldered to it already then) Solder the short pins to the screen, so the ones that are short from the black divider, making sure they are parallel to the screen or as much as they can as this might affect the fit of the whole thing (this is why its good that a lot of the time these screens come with the pins soldered to them)

  1. Put the screen on the custom PCB in the row of pins that are directly on top of the buttons, with the screen being on the same side as the buttons, so the long pins go into the board so that the pins are in the same area as the pins for the buttons and solder
  2. With the cutter, cut off the excess from putting on the screen, again as small as possible since the ESP32S2 will go on top of it
  3. Put the ESP32S2 on top of the pins that we had layed out towards the start, making sure that we have the custom PCB placed on a flat surface with the screen down on the surface, with this we place the ESP32S2 with the USB C connector on the left side of the custom PCB (following the lines that are layed out in the board essentially)
  4. Solder the board to the pins

With this we are done with the hardware side of the basic board and we can move on to the software and you may be wondering, shouldn't we be doing the battery? And the answer is NO! The reason for that is that we want to do software first as that way we can make sure that we have not burnt any connections when we were soldering and the only way to do this is through uploading the software! It is important that you do not connect the board to anything yet, for now leave it as is until it is mentioned that you can/should connect it to the computer

    Instructions for software!
  1. Download the code from the Github as a zip file and unzip it in a folder of your choosing. Say you chose to unzip inside of your documents folder, create a folder called Tamogotchi_Basic and move all the unzipped files inside, this is where our project will be hosted since that is the name of the folder that the project was created on and is needed to run the project from Arduino
  2. If you have not yet, install the Arduino IDE, just install everything that it chooses to install unless you know otherwise
  3. With the Arduino IDE open you want to go into the libraries section where we will download and install 2 Libraries
    • ArduinoJson library by Benoit Blanchon
    • TFT_eSPI library by Bodmer
  4. Go to the documents folder of your computer, go into libraries, go into TFT_eSPI, rename the "User_Setup.h" file to "User_Setup_Default.h" and from the github project copy the file that is named "User_Setup.h" onto that folder

  1. Go to OpenWeatherMap, create an account and once having done that, go to the top right of the screen where your username is, open the drop down menu and choose the My API Keys option. Once in that screen, generate a new API key and write it down somewhere (or just leave the window open)
  2. Open the Tamagotchi_Basic.ino and go to line 28 or 29 (the one that says "String openWeatherMapKey…") and on the string that is right after change the x's or value that is already there for the API key we got before

  3. Optional: The following are a couple of optional changes that could be done to the code so that the software is better suited to you
    • In line 33 or 34 is the configuration for wifi, so you can and should set it up here. If you are at Carleton College then this step is not needed as the settings that are there are those used for CarletonGuest, though you would need to set up network registration for the device

    • In line 193 or 194 the part that says "…=imperial…" can be changed for "…metric…" this makes it so that the weather is displayed in Celsius instead of Fahrenheit
    • In line 256 or 257 and 259 or 260 you can change the name of the device to whatever you want (in the case of the one shown at the start of this tutorial I changed it to Buddy-ogotchi for the person that I gifted that instance of it)

  1. Save the file
  2. Make sure that On button on the ESP32S2 board is placed correctly, if you placed the board correctly, assuming the screen is at the top (so furthest from you if you place it with the screen on a y axis) then the On button should be at the bottom, you can tell if it is because it does not have the letters RST below or above it, if the button is not there, then that means you put it on the wrong way, thus you should remove all the solder joints, turn it so that it is there (with the USB C connector at the top) and solder again.

  1. With Arudino IDE open, connect the USB Cable to your computer not to the board, press the On button on the board, you are going to want to keep holding this button as you plug in the cable to the board at least until you have verified the next step, the reason for that is that this action puts the board into download mode, which allows us to upload programs onto it!
  2. Once it is connected make sure that Arduino IDE has detected it (if you are on a Mac you might be asked if you want to connect to that, click yes), on the top left of the Arduino IDE you will see a dark drop down menu, it should have text that says some variation of ESP32S2 or Lolin S2 mini, etc, there are plenty options, but essentially you want the name of a board there and it should be some variation of the one we are using
  3. Click the upload button (the arrow that is to the left of where we checked for the board)
    4. Note: Once it finishes uploading an error might appear if it is "Upload error: Failed uploading: uploading error: exit status 1" that is fine and normal, you should not stress about it


We are now technically done with software, if you want to grab the MAC address for the board (for WI-FI purposes for example in the case of Carleton, where you need it for network registration) then you can go onto the Arduino terminal and scroll all the way up and MAC should be listed there.
However if you want to make sure (and you should make sure), click the RST button that is on the opposite side of the board from the On button, it says RST below it. If everything went on correctly then the screen will start displaying the background of the project along with the name that we chose for our Carl-o-Gotchi (essentially what is shown when I turn it on towards the end of the video at the start of this page), it should take a couple of seconds, if it takes a couple of minutes this is likely that the WI-FI is not properly set up as there is code that needs it so I would try to set it up again and download the code onto the board again.


If that does not work, you could try with a different OpenWeather API key as that might also be the case.
If that also does not work and it still takes minutes to boot up or it just gets stuck then I would remove the code that needs it, unfortunately at the time of writing this I am not aware of where that might be since I have not had the need for that yet (I have however had to set up WI-FI again and that is why I know that this is something that could happen)
NOW you have 2 choices:

  1. You can decide that this is it and you want to use this project with a cable connected to its side and follow the instructions that are further down in this page on how to put it inside the case
    2. OR
  2. You can do the fun part which is how I ended creating this tutorial which is adding A BATTERY!
